# Data Workbench 6.4 Release Notes{#data-workbench-release-notes}

## Atlas del dispositivo con cache in memoria{#device-atlas-with-in-memory-cache}

Il file DeviceAtlas.bundle ora utilizza una cache in memoria per migliorare notevolmente le prestazioni delle ricerche

Per impostazione predefinita, Device Atlas memorizzerà nella cache fino a 100.000 agenti utente e relative proprietà. La cache LRU è interamente indipendente all&#39;interno del DeviceAtlas.bundle, pertanto qualsiasi versione del server in grado di utilizzare un file bundle trarrà automaticamente vantaggio dalle prestazioni migliorate non appena il nuovo file verrà caricato.

**Modifica del file DeviceAtlas.cfg**

La dimensione massima della cache LRU può essere configurata modificando il parametro Limite elementi nella cache nel file DeviceAtlas.cfg.

A partire dalla versione Workbench dati 6.4, è possibile ignorare il limite predefinito di elementi nella cache di 100000 modificando il valore e salvando il file. Il valore predefinito è stato selezionato per essere sufficiente per la maggior parte delle esigenze.

```
component = DeviceAtlasComponent: 
  DeviceAtlas Bundle File = string: Lookups\\DeviceAtlas\\DeviceAtlas.bundle 
  Unsynchronized Bundle Extraction Path = string: Temp\\DeviceAtlas\\ 
  
<b> Cached Elements Limit = unsigned int: 100000</b> 
```

## Requisiti per l&#39;aggiornamento {#section-84f8f38e9c424d7788c79146fce5a6fe}

>[!IMPORTANT]
>
>È consigliabile utilizzare i file di configurazione predefiniti appena installati e personalizzarli, anziché spostare i file da un&#39;installazione precedente, con le seguenti eccezioni:

   * **`InsightServer64.exe`**
   * **`ReportServer.exe`**

## Aggiornamenti del sistema {#section-8e6b9dcaf333453bb2079b151bbbce93}

Queste funzioni sono state rinominate, eliminate oppure i file oppure le cartelle di installazione sono stati ristrutturati in questa versione:

* La cartella **Base.zip** non è più inclusa nel pacchetto di aggiornamento della versione.
* The **DeviceAtlas.bundle** file now uses an in-memory cache to improve the performance of lookups.
* Nel file **Log Processing.cfg** , il parametro *Chunk Size* in *Log Sources* è stato rimosso.
* Nel file **Disk Files.cfg** , il parametro *Detect Disk Corruption* è stato rimosso nei seguenti percorsi:

`\server\components\disk files.cfg`

`\server\components for processing servers\disk files.cfg`

* Nuove descrizioni dei servizi per *Adobe Analytics Premium Services* e per *Adobe Analytics Premium Report Services* nelle proprietà eseguibili.
* La funzione *Master Marketing Profile Export* (Esportazione profilo marketing principale) nella tabella dei dettagli è stata rinominata in ***Profili e pubblico Export***.
* La funzione *Test and Target Export* (Esportazione ***Test e Target) nella tabella Dettagli è stata rinominata in*** Adobe Target Export.

## Bug fissi {#section-bcdd732af53249b893d30bb42eeb14fc}

In Workbench dati 6.4 (dal rilascio di Workbench dati 6.31) sono state apportate le seguenti correzioni.

* Il punteggio di propensione non veniva reimpostato quando si eseguivano nuovamente input diversi nello stesso workspace. Adesso la reimpostazione avviene correttamente.
* Nessuna dimensione numerabile disponibile alla prima apertura della matrice di correlazione.
* L&#39;esportazione dei segmenti di Target non riusciva a causa dell&#39;assenza del campo mboxPC. Questo problema è stato adesso risolto.
* Richiesta dell&#39;ID formattata correttamente. L&#39;uso dell&#39;identificazione mbox3rdpartyId identification instead of default PCIDs caused Adobe Target to reject requests generated via the Target/Data Workbench integration (using the **`ExportIntegration.exe`**). La richiesta dell&#39;D viene adesso formattata correttamente e la velocità effettiva è corretta.
* La perdita di memoria del server di report durante l&#39;esportazione in Excel è stata risolta.

## Problemi noti {#section-d0cd72e82d1444a2b069dfcda5d7692a}

Di seguito sono riportate le limitazioni note in Workbench dati 6.4.

* **`ExportIntegration.exe`** richiede argomenti riga di comando in inglese. The output file name should be named in English for *Adobe Target Export*, *Profiles and Audiences Export*, and *Customer Record Service Export*.
* In *Profili e Audience Export*, l’immissione di caratteri non autorizzati (`[CR]` o `[TAB]`) come nomi di colonna genera registri non corretti che non consentono l’esportazione corretta dei dati.
* Nelle versioni cinese e giapponese, è possibile che nel browser Percorso si verificasse un problema di codifica dei caratteri Unicode.
